## Highest-priority items (start here)
1. **[HIGH] BuildMenuLock is dead** — `BuildMenuUI.RebuildVisibleRows` never checks the lock, so the tutorial's "Cabin/Torch/Bonfire only" restriction does nothing. (`audit-building-vendor.md`)
2. **[HIGH] StorageUI / FishStagingUI re-scan the whole scene every frame** while open (`FindObjectOfType` per slot). Real frame cost when a container is open. (`audit-ui.md`)
3. **[HIGH-latent] Deleted-model crash still reachable** — a restored save with `dirtyForExtraction=true` makes `AIMemoryExtractor` try to load the removed `.gguf`; the disable guard is missing on `EnsureModelLoadedAsync`/`OneShotAsync`. (`audit-ai-hal.md`)
4. **[MED] Held item + flashlight lost on load** from a piloted save (non-include-inactive lookups). (`audit-save-system.md`)
5. **[MED] NBodySimulation stale-body crash + NaN-at-r=0** — MissingReferenceException on a destroyed body; no gravity softening. (`audit-physics-world.md`)
6. **[MED] Dead enemies linger 30s** in the spawner list and starve MaxConcurrent. (`audit-combat-survival.md`)
7. **[MED] Camera.main NRE on fishing cast** — unguarded in `FishingRodController.SpawnBobber`. (`audit-portals-...md`)

## Cross-cutting themes (appear in many reports)
- **Runtime `Material` leaks** — new Material per-hit/per-frame never released: OceanMaskRenderer, PistolController tracer, MapOrbitLines, SpitProjectile, GhostPlacement, CombatFX. Cheap, widespread cleanup win.
- **Per-frame UI string allocation** — prompt strings rebuilt every frame without change-detection in several NPC/vendor scripts (a working change-detection pattern already exists in `BonfireInteraction.cs` — propagate it).
- **Dead code that's safe to delete** — entire LLM load path + IntentRouter + HAL tool actions (AI); dead `PackFloats`; `_TrailerSunAimer`; various write-only fields. See individual reports before deleting (some tutorial dead-code is retained for save compatibility — do NOT delete those).

## Reassuring results

- Save system apply-order matches docs and uses rb-based capture.
- The forbidden atmosphere/planet-generation zone was inspected read-only only — untouched.
- Recent fixes verified correct: crash-rebuild pickup/place (f1b9bde6), camera gating/throttle work.
