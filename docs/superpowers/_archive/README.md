# Archived design docs (historical)

These are the per-feature **specs** and **implementation plans** produced
during development (brainstorming → spec → plan → build cycles). They are
kept as a historical record of *why* and *how* each feature was built, but
they are **not** the source of truth for the current game.

For the current state of the project, see:

- `docs/CURRENT_STATE_AUDIT.md` — verified snapshot of every system + a scrap list
- `docs/GAME_OVERVIEW.md` — technical write-ups (short / medium / long)
- `CLAUDE.md` (repo root) — working guidance + the hard traps
- `docs/architecture-diagram.html` — interactive Mermaid architecture view

Dates in filenames are the day the doc was written.
